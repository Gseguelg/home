¿Qué es DPL?

El DIgSILENT Programming Language (DPL) es el lenguaje de scripting integrado en PowerFactory que permite automatizar cálculos, manipular la base de datos de la red eléctrica y controlar el flujo de trabajo de simulaciones. Funciona como un lenguaje procedimental orientado a objetos, donde las funciones se agrupan en cuatro categorías principales:

⚡
Funciones Globales
Funciones estáticas llamables desde cualquier punto del script. Gestión de proyectos, cálculos, archivos y sistema.
~70 funciones
🔧
Métodos Generales
Métodos disponibles para cualquier objeto de PowerFactory. Copia, navegación, conexiones, valores.
~50 métodos
🔌
Métodos Especializados
Métodos específicos de cada clase de objeto (Elm*, Sta*, Com*, Set*, Int*, Vis*, etc.).
+200 métodos
📋
Rutinas de Conjunto
Funciones para trabajar con el tipo de dato set, propio de DPL. Iteración, filtrado, ordenamiento.
19 rutinas
Nota: Las funciones marcadas con asterisco (*) en la documentación oficial también están disponibles en QDSL (Quasi Dynamic Simulation Language) y en los scripts de configuración de simulaciones RMS/EMT.

Sintaxis Básica

Acceso a atributos de objetos 
! Acceso a parámetro con el operador ':'
object load;
load = GetCalcRelevantObjects('*.ElmLod').First();
load:plini = 100.0;          ! Asignar valor de carga activa (MW)
double p = load:plini;       ! Leer valor

! Acceso a parámetro de resultado con 'c:'
object grid;
grid = GetSummaryGrid();
printf('Pérdidas: %f MW', grid:c:LossP);
Control de flujo
! Condicional
if (n > 0) {
  printf('positivo');
} else {
  printf('negativo o cero');
}

! Bucle while
int i = 0;
while (i < 10) {
  i += 1;
}

! Bucle for
int k;
for (k = 0; k < 5; k += 1) {
  printf('%d', k);
}

! break / continue / exit
exit();     ! Termina el script completamente

⚡ Funciones Globales

Funciones estáticas que pueden ser llamadas desde cualquier parte del script, sin necesidad de un objeto receptor. Abarcan gestión de proyectos, escenarios, cálculos, entorno y más.

Gestión de Proyectos y Casos de Estudio
ActivateProject(name) → int▶
int ActivateProject(string name)

Activa un proyecto por su nombre. Acepta nombre simple, nombre calificado (Project.IntPrj) o ruta completa (\User\Project.IntPrj).

Retorna: 0 en éxito, 1 si no se encuentra o no se puede activar.

🔧 Métodos Generales de Objeto

Estos métodos están disponibles en todos los objetos de PowerFactory. Se llaman con la sintaxis objeto.Método().

Creación, Copia y Eliminación
object.CreateObject(className [, nameParts...]) → object▶

Crea un nuevo objeto de la clase dada dentro del objeto receptor. Los nameParts se concatenan para formar el nombre.

object folder;
folder.CreateObject('ElmLod', 'Carga', 1);  ! Crea 'Carga1.ElmLod'
object.AddCopy(objectToCopy | set [, nameParts...]) → object▶

Agrega una copia de un objeto o conjunto al objeto receptor (destino). Respeta referencias internas entre objetos. Para copiar entre proyectos diferentes, usar PasteCopy().

object.PasteCopy(objects [, nameParts...]) → object▶

Similar a AddCopy, pero adapta automáticamente todas las referencias. Necesario cuando el objeto destino está en un proyecto diferente.

object.Move(targetFolder)

Mueve el objeto a la carpeta destino especificada.

object.CopyData(source)

Copia todos los parámetros (excepto loc_name y contenedores) del objeto fuente al objeto receptor.

Navegación y Contenido
object.GetContents([Name, recursive]) * → set▶

Retorna los objetos almacenados dentro del objeto. Name puede contener wildcards (*, ?). recursive=1 incluye sub-carpetas.

set lines;
lines = grid.GetContents('*.ElmLne', 1);  ! Todas las líneas recursivamente
object.GetChildren(hiddenMode [, filter, subfolders]) * → set▶

Como GetContents, pero también da acceso a objetos ocultos por la gestión de esquemas. hiddenMode: 0=solo visibles, 1=todos, 2=solo ocultos.

object.GetParent() * → object▶

Retorna la carpeta padre del objeto (mismo que el parámetro 'fold_id').

object.SearchObject(name) * → object▶

Busca un objeto por nombre dentro del objeto receptor. Soporta wildcards y filtros de clase.

object.GetReferences([filter, includeSubsets, includeHidden]) * → set▶

Retorna todos los objetos que tienen referencias al objeto receptor.

object.GetNet() → object (ElmNet)▶
